## Важно

Токен бота нельзя отправлять на GitHub.  
Файлы bot-token.txt и .env должны быть в .gitignore.

Пока проект маленький, всё оставляем в одном index.html. Разделять на css/js/backend лучше после теста на игроках или перед подключением серверного сохранения.

## Что делать следующим шагом

Следующий крупный этап — серверное сохранение и нормальная реферальная система:

1. Сделать backend.
2. Сохранять игрока по Telegram ID.
3. Сохранять money, totalEarned, upgrades, quests, stats в базе.
4. Сделать настоящие рефералки: кто кого пригласил.
5. Сделать рейтинг игроков.

До backend можно ещё улучшить:

- ежедневную цепочку наград;
- коллекции предметов;
- визуальные события;
- баланс первых 10–15 минут.

## Как обновлять игру

После изменений:

```bash
git add index.html DEV_NOTES.md
git commit -m "Update game"
git push
```

После push GitHub Pages обновляет игру примерно через 1–2 минуты.
